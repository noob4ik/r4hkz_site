# Первоначальная настройка ноды Meshtastic

Тут буду приводить названия настроек на английском, поскольку на русском они технически не грамотны.

Эти настройки отображаю мое видение правильных настроек для Самарской сети на начало зимы 2025.
## Ключевые настройки для работы в сети
### 1. Radio configuration - Lora 
Настройка Радиомодуля - Lora

- **Region:** Russia (соответствие требованиям ГРЧЦ)
- **Use preset:** on (ставим галочку)
- **Presets:** Long Range - Fast (настройки модуляции - все работаем на этом пресете, другие не услышат)
- **Ignore MQTT:** off (не игнорируем сообщения пришедшие через MQTT)
- **Ok to MQTT:** on (разрешаем пересылать свои сообщения в MQTT)
- **Transmit enabled:** on (но я бы сначала настроил имя ноды и только потом разрешил передачу в эфир)
- **Number of Hops:** 7 (ставим разрешенный максимум пока сеть малая)
- **Frequency slot:** 1 (номер канала - работаем на нем в Самаре)
- **RX Boosted Gain:** on (не у всех есть - это LNA по приему)
- **Frequency Override:** 868.825 (частота для радио модуля; обычно вычисляется из Region и Frequency slot автоматически, но можно указать вручную)

### 2. Device Configuration - User

**Внимание!** 
Эти настройки при первом выходе в эфир отправляются в сеть и именно под таким именем будут видеть другие участники в сети. 
Время от времени эта информация отправляется повторно (раз в 4-6 часов), до этого времени другие участники будут видеть ваше старое имя.

Рекомендую сначала настроить тут имя и только потом включать передачу в радио модуле.


- **Long Name:** это длинное название ноды в сети
- **Short Name:** 4 символа или 1 смайлик - короткое название, пишется в пиктограммах на карте, чатах
- **Unmessageable:** off (закроет возможность другим участникам сети писать на эту ноду личные сообщения; на этапе старта не рекомендую включать, принесет больше проблем чем пользы)
- **Licensed amateur radio (HAM):** off (**ни в коем случае не включать** - не позволит общаться в основных каналах, отключит большинство функций)

### 3. Device Configuration - Device

- Device Role:
    - **CLIENT_MUTE** - стандартная роль для портативных устройств или для устройств со слабой антенной - принимает/отправляет сообщения, не ретранслирует
    - **CLIENT** - базовая роль для ноды "на окне" - принимает/отправляет сообщения, ретранслирует сообщения других участников
    - **CLIENT_BASE** - продвинутая роль для ноды "на крыше" - принимает/отправляет сообщения, ретранслирует сообщения других участников; сообщения нод добавленных на этой ноде в избранное - ретранслирует в приоритете и может переслать если лимит прыжков исчерпан
- **Rebroadcast Mode:** ALL - (ретранслируем все сообщения в сети)
